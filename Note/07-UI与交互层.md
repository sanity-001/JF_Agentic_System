## 2. 三种运行模式

### 2.1 交互式 TUI（默认）

```bash
oh                    # 启动 React/Ink 界面
```

内部调用链：

```
cli.py: main()
  └─→ asyncio.run(run_repl(prompt=None, ...))
        └─→ launch_react_tui(...)          # ui/react_launcher.py
              └─→ 启动 Node.js 子进程运行 frontend/terminal/
                    通过 stdin/stdout JSON-lines 协议通信
```

### 2.2 单次问答（-p）

```bash
oh -p "Explain this codebase"                     # 文本输出
oh -p "List all functions" --output-format json    # JSON 输出
oh -p "Fix the bug" --output-format stream-json    # 流式 JSON
```

内部调用链：

```
cli.py: main()
  └─→ asyncio.run(run_print_mode(prompt=..., output_format=...))
        └─→ handle_line(prompt, runtime_bundle)
              └─→ engine.run_query(...)
                    └─→ 收集所有 StreamEvent，按格式输出
```

### 2.3 Dry-Run（安全预览）

```bash
oh --dry-run -p "Review this bug fix"
```

不启动模型，不连接 MCP，只做静态配置解析和输出。

## 3. React/Ink TUI 通信协议

### 3.1 协议设计

Python 后端和 Node.js 前端通过 **JSON-lines 协议** 在 stdin/stdout 上通信：

```
格式: "OHJSON:" + JSON字符串 + "\n"

Python 后端 → Node 前端的事件:
  OHJSON:{"type":"assistant_text_delta","text":"Hello"}\n
  OHJSON:{"type":"tool_execution_started","tool_name":"Read"}\n
  OHJSON:{"type":"tool_execution_completed","tool_name":"Read"}\n
  OHJSON:{"type":"permission_prompt","tool_name":"Write",...}\n

Node 前端 → Python 后端的请求:
  OHJSON:{"type":"user_input","text":"fix the bug"}\n
  OHJSON:{"type":"permission_response","approved":true}\n
  OHJSON:{"type":"slash_command","command":"model"}\n
```

## 4. Build Runtime —— 运行时的依赖注入

```python
# ui/runtime.py → build_runtime()
# 这是整个系统的"组装工厂"

async def build_runtime(
    cwd, model, max_turns, base_url, system_prompt,
    api_key, api_format, permission_mode, effort,
) -> RuntimeBundle:
    
    # 1. 加载配置
    settings = load_settings().merge_cli_overrides(...)
    
    # 2. 解析 Provider + 认证
    api_client = _resolve_api_client_from_settings(settings)
    
    # 3. 构建核心组件
    tool_registry = create_default_tool_registry()
    permission_checker = PermissionChecker(settings.permission)
    hook_executor = HookExecutor(...)
    
    # 4. 加载技能和插件
    skill_registry = load_skill_registry(cwd, settings=settings)
    plugins = load_plugins(settings, cwd)
    
    # 5. 构建系统提示词
    system_prompt = build_runtime_system_prompt(settings, cwd, ...)
    
    # 6. 创建查询引擎
    engine = QueryEngine(
        api_client=api_client,
        tool_registry=tool_registry,
        permission_checker=permission_checker,
        cwd=cwd,
        model=model,
        system_prompt=system_prompt,
        ...
    )
    
    # 7. 返回打包好的 RuntimeBundle
    return RuntimeBundle(
        engine=engine,
        tool_registry=tool_registry,
        permission_checker=permission_checker,
        skill_registry=skill_registry,
        plugins=plugins,
        settings=settings,
        ...
    )
```

`RuntimeBundle` 被传入 `ReactBackendHost`，作为整个 TUI 会话的共享状态。

## 5. 输出格式对比

### 5.1 Text（默认）

```
oh -p "Explain this file" --output-format text
```

输出：纯文本 Markdown，适合人类阅读。

### 5.2 JSON


输出：结构化 JSON，适合脚本解析：

```json

  "result": "...",
  "usage": {
    "input_tokens": 1500,
    "output_tokens": 300,
    "cost_usd": 0.015
  },
  "tool_calls": [...]
}
```

### 5.3 Stream-JSON

```
oh -p "Fix the bug" --output-format stream-json
```

输出：逐行 JSON 事件流，适合实时管道处理：

```json

{"type":"assistant_text_delta","text":"read the file"}
{"type":"tool_execution_started","tool_name":"Read"}
{"type":"tool_execution_completed","tool_name":"Read"}
{"type":"assistant_text_delta","text":"The bug is on line 42"}
{"type":"assistant_turn_complete","usage":{...}}
```

## 6. Task Worker 模式

`--task-worker` 是隐藏参数，用于后台 Agent 工作进程：

```bash
oh --task-worker   # 从 stdin 读取任务，后台执行
```

这是多 Agent 协调的基础——每个子 Agent 都以 `--task-worker` 模式启动，通过 stdin 接收指令，通过 stdout 输出结果。

---

## 7. Backend-Only 模式

`--backend-only` 也是隐藏参数，用于 **分离前端和后端进程**：

```bash
oh --backend-only   # 仅启动 Python 后端（不启动 Node.js 前端）
```

React TUI 前端会自行启动这个后端进程并接管 stdin/stdout。

## 10. UI 设计哲学

| 原则 | 实现 |
|------|------|
| **前后端分离** | Python 引擎 + React/Ink 前端，通过协议解耦 |
| **流式优先** | 所有模型输出以流式事件推送，不等待完整响应 |
| **低延迟反馈** | Spinner 动画、工具状态实时更新 |
| **跨平台** | React/Ink 抽象了终端差异，Windows/macOS/Linux 一致体验 |
| **降级可用** | 没有 Node.js 时回退到纯文本交互（questionary 库） |
| **非交互模式** | 支持管道和脚本（text/json/stream-json） |
