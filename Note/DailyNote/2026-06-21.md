## 二、Agent Plugin 加载 Bug 修复

### 2.1 相对导入失败（核心 Bug）
**问题**：`from . import BASE_URL, get_session` 在 OpenHarness 的动态加载机制下失败——`importlib` 不设置 `__package__`。

**修复**：5 个工具文件全部内联共享代码（BASE_URL + _get_session），去掉相对导入。

### 2.2 缺少 input_model 属性
**问题**：`BaseTool` 要求 `input_model: type[BaseModel]` 生成 JSON Schema，37 个 Tool 全都没设。

**修复**：每个 Tool 类加 `input_model = <对应InputClass>`。

### 2.3 函数命名混乱
**问题**：linter 改函数名为 `___get_session`（三下划线），调用还是 `__get_session`（双下划线），Python name mangling 导致 `_ClassName__get_session` 找不到。

**修复**：统一为 `_get_session`（单下划线，不会 name mangling）。

### 2.4 Plugin 加载配置
- `allow_project_plugins` 默认 `false` → 需在 `~/.openharness/settings.json` 启用
- 更推荐：`ln -s` 到 `~/.openharness/plugins/` 作为全局插件

## 三、后端 Bug 修复

### 3.1 load_config_file 未启动 Receiver
**问题**：`connect()` 会先启动 receiver，但 `load_config_file()` 没这步。

**修复**：`load_config_file` 开头加 `if not self.receiver_running: self.start_receiver()`

### 3.2 水冷机状态字段名不一致
**问题**：Tool 检查 `data.get("connected")`，但真实 MODBUS 模式返回 `"connection": "connected"`。

**修复**：`chiller_get_status` 和 `chiller_wait_stable` 同时检查两种字段。

### 3.3 缺失 chiller_connect/chiller_disconnect 工具
新增两个 Tool：`chiller_connect`（默认 `/dev/ttyUSB1`，4800 baud，slave 1）和 `chiller_disconnect`。

## 四、Linux 部署问题

### 4.1 system_startup 路径问题
- `conda run` 在 subprocess 中找不到 conda → 改用 conda 环境的 Python 可执行文件路径
- `~/miniconda3/envs/slsdet9/bin/python start.py`
- 支持 `JF_CONDA_PYTHON` 环境变量覆盖

### 4.2 slsReceiver 找不到
**问题**：子进程 PATH 不含 conda bin 目录。

**修复**：`system_startup` 启动子进程时把 `_CONDA_BIN` 加到 PATH。

### 4.3 串口权限
**问题**：`Permission denied: /dev/ttyUSB0`

**解决**：`sudo usermod -a -G dialout $USER` + 重新登录。

### 4.4 串口设备号变化
**问题**：服务器重启后水冷机从 `/dev/ttyUSB0` 变成 `/dev/ttyUSB1`。

**修复**：Agent 和前端默认端口全部改为 `/dev/ttyUSB1`。

### 4.5 端口占用
**问题**：`[Errno 98] Address already in use` — 旧进程残留。

**解决**：`fuser -k 8000/tcp`。
