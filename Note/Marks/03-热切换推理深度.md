# Q3: 交互式会话中如何热切换推理深度

## 问题

在 `oh` 交互式会话中如何切换推理深度（effort）？

## 解答

直接输入斜杠命令，**不需要退出会话**：

```
/effort low       ← 最快、最便宜
/effort medium    ← 均衡
/effort high      ← 深度推理
/effort xhigh     ← 最强（/effort max 等价）
/effort show      ← 查看当前设置
```

## 源码解析

`commands/registry.py` 第 1232-1254 行：

```python
async def _effort_handler(args: str, context: CommandContext) -> CommandResult:
    value = args.strip()
    if value == "max":
        value = "xhigh"

    # ① 持久化到 settings.json
    settings.effort = value
    save_settings(settings)

    # ② 热切换引擎状态（关键！）
    context.engine.set_effort(value)

    # ③ 重建系统提示词
    context.engine.set_system_prompt(
        build_runtime_system_prompt(settings, cwd=context.cwd, ...)
    )

    # ④ 同步 UI 状态栏
    context.app_state.set(effort=value)
```

## 热切换链路

```
/effort high
  → engine.set_effort("high")      # 修改 self._effort
  → engine.set_system_prompt(...)  # 重建提示词
  → app_state.set(effort="high")   # UI 立即显示
  → 下一次 run_query() 自动使用新 effort
```

## 所有可热切换的命令

| 命令 | 作用 | 调用的引擎方法 |
|------|------|--------------|
| `/effort` | 推理深度 | `engine.set_effort()` |
| `/model sonnet` | 切换模型 | `engine.set_model()` |
| `/turns 10` | 最大轮次 | `engine.set_max_turns()` |
| `/permissions` | 权限模式 | `engine.set_permission_checker()` |
| `/provider use` | 切换后端 | `engine.set_api_client()` |
| `/fast` | 快速模式 | 修改 settings + UI |
