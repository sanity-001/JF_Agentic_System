# Q5: _stream_compaction 与 _preprocess_images 的原理

## 问题

Agent Loop 中这两个预处理步骤的原理是什么？

## 解答

两者都在模型调用前处理 `messages`，确保传给 API 的消息合法。

---

## 一、`_stream_compaction` — 对话压缩

### 为什么需要

Agent 会话可能 50+ 轮，累积数万字。模型有上下文窗口限制（200K tokens），超出 API 直接报错。

### 四级压缩策略

```
超出阈值？
  ├─ Level 1: Microcompact（微型压缩）
  │    原理：清除旧工具输出，替换为 "[Old tool result content cleared]"
  │    成本：0 token（不调用 LLM）
  │    速度：毫秒级
  │    代码：compact/__init__.py microcompact_messages()
  │
  ├─ Level 2: Context Collapse
  │    原理：长文本首尾拼接，中间截断
  │    成本：0 token
  │
  ├─ Level 3: Session Memory
  │    原理：历史消息浓缩为摘要行
  │    成本：0 token
  │
  └─ Level 4: Full Compaction
       原理：调用 LLM 生成结构化摘要
       成本：一次 API 调用
       速度：数秒
```

### Microcompact 核心代码

```python
def microcompact_messages(messages, *, keep_recent=5):
    all_ids = _collect_compactable_tool_ids(messages)
    if len(all_ids) <= keep_recent:
        return messages, 0   # 不够多，不需要清理

    keep_set = set(all_ids[-keep_recent:])  # 保留最近 5 个
    clear_set = set(all_ids) - keep_set     # 其余清除

    for msg in messages:
        for block in msg.content:
            if block.tool_use_id in clear_set:
                # 5000 字文件内容 → 40 字占位文本
                block.content = "[Old tool result content cleared]"
```

核心思路：旧工具结果占最多 token，但保留工具调用结构就够了。

## 二、`_preprocess_images_in_messages` — 图片转文本

### 为什么需要

- 多模态模型（claude-sonnet-4-6）：直接传图片 → 跳过
- 纯文本模型（deepseek-chat）：传图片会报错 → 必须先转成文字描述

### 核心代码

```python
async def _preprocess_images_in_messages(messages, context):
    # 1. 模型支持多模态？直接跳过
    if is_model_multimodal(context.model):
        return

    # 2. 扫描所有消息中的 ImageBlock
    pending = []
    for msg_idx, msg in enumerate(messages):
        for blk_idx, block in enumerate(msg.content):
            if isinstance(block, ImageBlock):
                pending.append((msg_idx, blk_idx, block))

    if not pending:
        return

    # 3. 并行调用 image_to_text 工具将图片转为文本
    results = await asyncio.gather(
        *[_describe(mi, bi, blk) for mi, bi, blk in pending]
    )

    # 4. 原地替换：ImageBlock → TextBlock
    for msg_idx, blk_idx, description in results:
        msg = messages[msg_idx]
        msg.content[blk_idx] = TextBlock(text=description)
        # 原来: ImageBlock(base64 图片数据)
        # 现在: TextBlock("图片显示了一个终端窗口...")
```

### 两者对比

| | _stream_compaction | _preprocess_images |
|---|---|---|
| **解决问题** | 对话太长超出上下文 | 纯文本模型看不懂图片 |
| **触发条件** | 每次调用前检查 token 数 | 模型不支持多模态 + 消息中有图片 |
| **核心操作** | 删除/摘要旧消息 | 调用视觉模型转文字 |
| **是否调用 LLM** | Level 1-3 不需要，Level 4 需要 | 需要 |
| **修改对象** | 工具结果 → 占位文本 | ImageBlock → TextBlock |
